## Why

GitHub Actions and local development should target the current runtime and package-manager versions the project intends to support. Moving to Node 24 and pnpm v11 keeps the action metadata, CI workflow, manifest constraints, lockfile, and documentation aligned around one supported toolchain.

## What Changes

- **BREAKING**: Update the GitHub Action runtime from `node20` to `node24`, requiring runners that support the Node 24 action runtime.
- Update project package-manager metadata from pnpm 9 to pnpm 11.
- Update engine requirements and documented prerequisites to require Node 24 and pnpm 11.
- Update repository workflows to install and test/build with Node 24 and pnpm 11.
- Refresh generated artifacts affected by the toolchain change, including the pnpm lockfile and committed action bundle if the build output changes.
- Non-goals: do not change bundle-size comparison behavior, threshold enforcement, PR comments, tarball URI semantics, or target project build orchestration.

## Capabilities

### New Capabilities
- `runtime-toolchain`: Defines the supported GitHub Action runtime and repository package-manager/toolchain requirements.

### Modified Capabilities

None.

## Impact

- `action.yml` changes the GitHub Actions runtime to Node 24.
- `package.json` updates `engines` and `packageManager` for Node 24 and pnpm 11.
- `pnpm-lock.yaml` may be regenerated by pnpm v11.
- `.github/workflows/bundle-size.yml` runs setup and checks using Node 24 and pnpm 11.
- `README.md` updates runtime, prerequisites, and workflow examples.
- `dist/index.js` and `dist/licenses.txt` must remain synchronized after rebuilding the committed action bundle.
- Tests and build verification should run with pnpm v11 on Node 24.
